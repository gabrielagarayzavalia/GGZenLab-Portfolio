1. Regla de oro

ConceptoRegla
mainSiempre estable; solo entra por PR merge
Ramasfeature/ fix/ chore/ wip/
ProhibidoPush directo a main; reset --hard sin rama backup
AgentesNunca commitear en main
  main (protegido)  ◄────  merge PR  ◄────  feature/fix branch
       ▲                                        │
       └──────── git pull (post-merge) ──────────┘

2. Convención de ramas

PrefijoUsoEjemplo
feature/Funcionalidad nuevafeature/qa-job-hunter-desktop-launcher
fix/Bugfix/qa-job-hunter-env-y-shortcut
hotfix/Urgente en mainhotfix/dashboard-crash
chore/Tooling, docschore/docs-guia-12-pasos
wip/Experimento agentewip/spike-ui
rescue/Recuperar reflogrescue/commits-perdidos

3. Flujo diario

  main ──git switch -c──► feature/fix ──commits──► push ──► PR ──merge──► main
                                                              │
                                                    git pull ◄┘

git checkout main && git pull origin main
git switch -c feature/mi-tarea

CampoValor

10. Mapa "¿dónde estoy?"

  En main:     * main  →  igual que GitHub tras git pull
  En feature:  * feature/x  →  main NO tiene tus commits hasta merge PR
  Tras checkout main:  git pull origin main  (siempre)

12. Checklist cierre sesión

  • Rama feature/fix (no main)
  • Push hecho
  • PR creado o mergeado
  • Secretos solo local
  • git pull en main tras merge
