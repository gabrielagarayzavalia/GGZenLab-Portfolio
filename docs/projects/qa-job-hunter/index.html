Mini-project: QA Job Hunter — Mongo persistence lab

Local MongoDB persistence for job match results: Docker compose, seed from JSON, Gherkin acceptance criteria, and API tests for GET /api/jobs.

Scope (US-JH-B06-1 + US-JH-B06-4)

  • Docker mongo:7 with healthcheck
  • Collections: analysis_runs, jobs, skipped_jobs
  • Seed: npm run db:seed from output/jobs-result.json
  • API: GET /api/jobs?sort=matchPercent&order=desc

QA artifacts in the repo

  • Gherkin: projects/qa-job-hunter/gherkin/mongo-persistence.feature
  • API tests: projects/qa-job-hunter/tests/api/jobs.test.ts
  • Compose: projects/qa-job-hunter/docker-compose.yml
  • Product code: projects/qa-job-hunter/src/db/

Quick run

cd projects/qa-job-hunter
docker compose up -d
npm install
npm run db:seed
npm run dashboard
npm run test:api

Requires output/jobs-result.json from a prior analyze run. API tests skip gracefully if the dashboard is not running.
